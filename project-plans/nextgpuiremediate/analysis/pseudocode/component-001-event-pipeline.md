# Component 001: Event Pipeline

Plan ID: PLAN-20260219-NEXTGPUIREMEDIATE
Component: Event Pipeline
Created: 2026-02-19

---

## Overview

This component defines the core event pipeline infrastructure that routes all events between the GPUI UI layer (smol runtime) and the business logic layer (tokio runtime). The pipeline consists of: EventBus, GpuiBridge, user_event_forwarder, and ViewCommandSink.

---

## Requirement Coverage

### REQ-EVT-001: EventBus Broadcast

**REQ-EVT-001.1**: EventBus MUST deliver events to all active subscribers

- **Full Text**: The EventBus must use a broadcast channel pattern where any event emitted is delivered to all currently active subscribers. No subscriber should miss events while actively subscribed.
- **GIVEN**: An EventBus with N active subscribers
- **WHEN**: An AppEvent is emitted via emit()
- **THEN**: All N subscribers receive the event via their receiver channels
- **Why**: Decouples components - any component can react to any event without direct coupling to the emitter

### REQ-EVT-001.2: Events MUST be delivered in emission order per subscriber

- **Full Text**: For a single subscriber, events must arrive in the same order they were emitted. This ensures predictable state transitions.
- **GIVEN**: Events A, B, C emitted in that order
- **WHEN**: A subscriber receives events
- **THEN**: Events arrive in order A, B, C (not reordered)
- **Why**: State machines in presenters depend on ordered events (e.g., StreamStarted before TextDelta)

### REQ-EVT-001.3: Slow subscribers MUST NOT block fast subscribers

- **Full Text**: The broadcast channel must not have a single slow consumer blocking all other consumers. Each subscriber has independent receive capacity.
- **GIVEN**: Subscriber A is slow (not calling recv), Subscriber B is fast
- **WHEN**: Events are emitted continuously
- **THEN**: Subscriber B continues to receive events normally
- **Why**: Prevents a misbehaving presenter from freezing the entire UI

### REQ-EVT-001.4: Lagged subscribers MUST receive lagged notification

- **Full Text**: When a subscriber falls behind (channel buffer overflow), they must receive a RecvError::Lagged notification with the count of missed events.
- **GIVEN**: A subscriber that hasn't called recv() for a while
- **WHEN**: Channel buffer fills and overflows
- **THEN**: Next recv() returns Err(RecvError::Lagged(count))
- **Why**: Allows presenters to recover gracefully (e.g., refresh full state)

### REQ-EVT-002: Runtime Bridge

**REQ-EVT-002.1**: GpuiBridge MUST forward UserEvents from GPUI to tokio EventBus

- **Full Text**: User events generated in GPUI views (running on smol) must be forwarded to the tokio-based EventBus so presenters can handle them.
- **GIVEN**: A UserEvent emitted in a GPUI view
- **WHEN**: The bridge's user_event_forwarder is running
- **THEN**: The event appears on the tokio EventBus as AppEvent::User(...)
- **Why**: Bridges the runtime gap between UI (smol) and business logic (tokio)

**REQ-EVT-002.2**: GpuiBridge MUST deliver ViewCommands from tokio to GPUI

- **Full Text**: ViewCommands generated by presenters (tokio) must be delivered to GPUI views so they can update their state and re-render.
- **GIVEN**: A ViewCommand sent by a presenter via ViewCommandSink
- **WHEN**: GPUI's event loop polls for commands
- **THEN**: The view receives the command and updates its state
- **Why**: Allows presenters to control view state without direct coupling

**REQ-EVT-002.3**: Bridge MUST handle runtime shutdown gracefully

- **Full Text**: When the application shuts down, the bridge must stop cleanly without panics, dropped events, or hung tasks.
- **GIVEN**: Application shutdown is initiated
- **WHEN**: GpuiBridge receives shutdown signal
- **THEN**: All channels close cleanly, tasks terminate, no panic
- **Why**: Clean shutdown prevents data corruption and resource leaks

### REQ-EVT-003: Event Logging

**REQ-EVT-003.1**: All events MUST be logged with structured tracing

- **Full Text**: Every event emission must produce a structured log entry for debugging and auditing.
- **GIVEN**: Any AppEvent emitted
- **WHEN**: emit() or emit_with_context() is called
- **THEN**: A tracing::debug! log is recorded with event details
- **Why**: Essential for debugging event flow issues

**REQ-EVT-003.2**: Logs MUST include event type name and payload

- **Full Text**: Log entries must contain the discriminant name (e.g., "UserEvent::SendMessage") and the event's Debug representation.
- **GIVEN**: AppEvent::User(UserEvent::SendMessage { text: "hi" })
- **WHEN**: Logged
- **THEN**: Log contains event_type="UserEvent::SendMessage" event=SendMessage { text: "hi" }
- **Why**: Makes logs grep-able by event type

**REQ-EVT-003.3**: Sensitive data in events MUST be redacted in logs

- **Full Text**: API keys, tokens, and other sensitive fields must be masked or omitted in log output.
- **GIVEN**: An event containing api_key field
- **WHEN**: Logged
- **THEN**: api_key value is "[REDACTED]" or omitted
- **Why**: Security - prevents credentials in log files

## Pseudocode

### EventBus Implementation

```pseudocode
001: MODULE EventBus
002: 
003: STRUCT EventBus
004:   sender: broadcast::Sender<AppEvent>
005:   capacity: usize
006: END STRUCT
007: 
008: // REQ-EVT-001.1: Create EventBus with broadcast channel
009: FUNCTION new(capacity: usize) -> EventBus
010:   LET (sender, _) = broadcast::channel(capacity)
011:   RETURN EventBus { sender, capacity }
012: END FUNCTION
013: 
014: // REQ-EVT-001.1, REQ-EVT-003.1, REQ-EVT-003.2: Emit event to all subscribers
015: FUNCTION emit(self, event: AppEvent)
016:   // REQ-EVT-003.1: Log event with structured tracing
017:   LET event_type = event.type_name()
018:   // REQ-EVT-003.3: Redact sensitive fields
019:   LET safe_event = redact_sensitive(event.clone())
020:   tracing::debug!(
021:     event_type = %event_type,
022:     event = ?safe_event,
023:     "Event emitted"
024:   )
025:   
026:   // REQ-EVT-001.1: Broadcast to all subscribers
027:   // REQ-EVT-001.3: send() is non-blocking, doesn't wait for slow receivers
028:   MATCH self.sender.send(event)
029:     Ok(receiver_count) => {
030:       tracing::trace!("Event delivered to {} subscribers", receiver_count)
031:     }
032:     Err(_) => {
033:       // No active subscribers - this is OK
034:       tracing::trace!("Event emitted but no subscribers")
035:     }
036:   END MATCH
037: END FUNCTION
038: 
039: // REQ-EVT-003.1: Emit with source context for debugging
040: FUNCTION emit_with_context(self, event: AppEvent, source: &str)
041:   LET event_type = event.type_name()
042:   LET safe_event = redact_sensitive(event.clone())
043:   tracing::debug!(
044:     event_type = %event_type,
045:     event = ?safe_event,
046:     source = %source,
047:     "Event emitted with context"
048:   )
049:   
050:   LET _ = self.sender.send(event)
051: END FUNCTION
052: 
053: // REQ-EVT-001.1: Create new subscription
054: FUNCTION subscribe(self) -> broadcast::Receiver<AppEvent>
055:   // REQ-EVT-001.2: Each receiver gets events in order
056:   // REQ-EVT-001.4: Receiver will get Lagged error if too slow
057:   RETURN self.sender.subscribe()
058: END FUNCTION
059: 
060: // REQ-EVT-003.3: Helper to redact sensitive data
061: FUNCTION redact_sensitive(event: AppEvent) -> AppEvent
062:   MATCH event
063:     User(SaveProfile { profile }) => {
064:       LET mut safe_profile = profile.clone()
065:       IF safe_profile.api_key.is_some() THEN
066:         safe_profile.api_key = Some("[REDACTED]".to_string())
067:       END IF
068:       RETURN User(SaveProfile { profile: safe_profile })
069:     }
070:     User(SaveMcpConfig { config }) => {
071:       // Redact any secrets in MCP config
072:       LET safe_config = redact_mcp_config(config)
073:       RETURN User(SaveMcpConfig { config: safe_config })
074:     }
075:     _ => RETURN event  // No sensitive data
076:   END MATCH
077: END FUNCTION
078: 
079: END MODULE
```

### GpuiBridge Implementation

```pseudocode
080: MODULE GpuiBridge
081: 
082: STRUCT GpuiBridge
083:   user_event_tx: async_channel::Sender<UserEvent>
084:   user_event_rx: async_channel::Receiver<UserEvent>
085:   view_command_tx: async_channel::Sender<ViewCommand>
086:   view_command_rx: async_channel::Receiver<ViewCommand>
087:   shutdown: Arc<AtomicBool>
088: END STRUCT
089: 
090: // REQ-EVT-002.1, REQ-EVT-002.2: Create bridge with bidirectional channels
091: FUNCTION new() -> GpuiBridge
092:   LET (user_event_tx, user_event_rx) = async_channel::bounded(256)
093:   LET (view_command_tx, view_command_rx) = async_channel::bounded(256)
094:   LET shutdown = Arc::new(AtomicBool::new(false))
095:   
096:   RETURN GpuiBridge {
097:     user_event_tx,
098:     user_event_rx,
099:     view_command_tx,
100:     view_command_rx,
101:     shutdown,
102:   }
103: END FUNCTION
104: 
105: // REQ-EVT-002.1: Called by GPUI views to emit UserEvent
106: FUNCTION emit_user_event(self, event: UserEvent)
107:   tracing::trace!("GpuiBridge: emitting UserEvent {:?}", event)
108:   // Non-blocking send to channel (GPUI/smol side)
109:   IF self.user_event_tx.try_send(event).is_err() THEN
110:     tracing::warn!("User event channel full, event dropped")
111:   END IF
112: END FUNCTION
113: 
114: // REQ-EVT-002.2: Get ViewCommandSink for presenters to send commands
115: FUNCTION view_command_sink(self) -> ViewCommandSink
116:   RETURN ViewCommandSink::new(self.view_command_tx.clone())
117: END FUNCTION
118: 
119: // REQ-EVT-002.2: Called by GPUI to poll for view commands
120: FUNCTION try_recv_view_command(self) -> Option<ViewCommand>
121:   MATCH self.view_command_rx.try_recv()
122:     Ok(cmd) => RETURN Some(cmd)
123:     Err(_) => RETURN None
124:   END MATCH
125: END FUNCTION
126: 
127: // REQ-EVT-002.3: Signal shutdown
128: FUNCTION shutdown(self)
129:   self.shutdown.store(true, Ordering::SeqCst)
130:   // Close channels to unblock any waiting tasks
131:   self.user_event_tx.close()
132:   self.view_command_rx.close()
133:   tracing::info!("GpuiBridge shutdown signaled")
134: END FUNCTION
135: 
136: END MODULE
```

### User Event Forwarder

```pseudocode
137: MODULE UserEventForwarder
138: 
139: // REQ-EVT-002.1: Spawn task that forwards UserEvents to EventBus
140: FUNCTION spawn_user_event_forwarder(
141:   user_event_rx: async_channel::Receiver<UserEvent>,
142:   event_bus: Arc<EventBus>,
143:   shutdown: Arc<AtomicBool>
144: ) -> JoinHandle<()>
145:   
146:   RETURN tokio::spawn(async move {
147:     tracing::info!("User event forwarder started")
148:     
149:     LOOP
150:       // REQ-EVT-002.3: Check shutdown flag
151:       IF shutdown.load(Ordering::SeqCst) THEN
152:         tracing::info!("User event forwarder shutting down")
153:         BREAK
154:       END IF
155:       
156:       // Wait for next user event from GPUI bridge
157:       MATCH user_event_rx.recv().await
158:         Ok(user_event) => {
159:           tracing::trace!("Forwarding UserEvent: {:?}", user_event)
160:           // REQ-EVT-002.1: Wrap and emit to EventBus
161:           LET app_event = AppEvent::User(user_event)
162:           event_bus.emit(app_event)
163:         }
164:         Err(RecvError) => {
165:           // Channel closed, shut down
166:           tracing::info!("User event channel closed, forwarder stopping")
167:           BREAK
168:         }
169:       END MATCH
170:     END LOOP
171:     
172:     tracing::info!("User event forwarder stopped")
173:   })
174: END FUNCTION
175: 
176: END MODULE
```

### ViewCommandSink Implementation

```pseudocode
177: MODULE ViewCommandSink
178: 
179: STRUCT ViewCommandSink
180:   sender: async_channel::Sender<ViewCommand>
181:   notifier: Option<Arc<Notify>>  // Optional: notify GPUI of new command
182: END STRUCT
183: 
184: // REQ-EVT-002.2: Create ViewCommandSink
185: FUNCTION new(sender: async_channel::Sender<ViewCommand>) -> ViewCommandSink
186:   RETURN ViewCommandSink { sender, notifier: None }
187: END FUNCTION
188: 
189: // REQ-EVT-002.2: Create with notifier for immediate GPUI wakeup
190: FUNCTION with_notifier(
191:   sender: async_channel::Sender<ViewCommand>,
192:   notifier: Arc<Notify>
193: ) -> ViewCommandSink
194:   RETURN ViewCommandSink { sender, notifier: Some(notifier) }
195: END FUNCTION
196: 
197: // REQ-EVT-002.2: Send single ViewCommand to GPUI
198: FUNCTION send(self, command: ViewCommand)
199:   tracing::trace!("Sending ViewCommand: {:?}", command)
200:   
201:   MATCH self.sender.try_send(command)
202:     Ok(()) => {
203:       // Notify GPUI if configured
204:       IF let Some(notifier) = &self.notifier THEN
205:         notifier.notify_one()
206:       END IF
207:     }
208:     Err(TrySendError::Full(_)) => {
209:       tracing::warn!("ViewCommand channel full, command queued")
210:       // Use blocking send as fallback
211:       LET _ = self.sender.send(command).await
212:     }
213:     Err(TrySendError::Closed(_)) => {
214:       tracing::error!("ViewCommand channel closed, command dropped")
215:     }
216:   END MATCH
217: END FUNCTION
218: 
219: // REQ-EVT-002.2: Send batch of ViewCommands efficiently
220: FUNCTION send_batch(self, commands: Vec<ViewCommand>)
221:   FOR command IN commands DO
222:     MATCH self.sender.try_send(command)
223:       Ok(()) => continue
224:       Err(_) => tracing::warn!("Batch command dropped")
225:     END MATCH
226:   END FOR
227:   
228:   // Single notification for entire batch
229:   IF let Some(notifier) = &self.notifier THEN
230:     notifier.notify_one()
231:   END IF
232: END FUNCTION
233: 
234: // REQ-EVT-002.2: Async send that waits if channel is full
235: ASYNC FUNCTION send_async(self, command: ViewCommand)
236:   tracing::trace!("Async sending ViewCommand: {:?}", command)
237:   
238:   MATCH self.sender.send(command).await
239:     Ok(()) => {
240:       IF let Some(notifier) = &self.notifier THEN
241:         notifier.notify_one()
242:       END IF
243:     }
244:     Err(_) => {
245:       tracing::error!("ViewCommand channel closed")
246:     }
247:   END MATCH
248: END FUNCTION
249: 
250: END MODULE
```

### Presenter Event Loop Pattern

```pseudocode
251: MODULE PresenterEventLoop
252: 
253: // REQ-EVT-001.1, REQ-EVT-001.4: Standard presenter event loop pattern
254: FUNCTION run_presenter_event_loop<P: Presenter>(
255:   presenter: Arc<P>,
256:   event_bus: Arc<EventBus>,
257:   shutdown: CancellationToken
258: ) -> JoinHandle<()>
259:   
260:   RETURN tokio::spawn(async move {
261:     // REQ-EVT-001.1: Subscribe to all events
262:     LET mut receiver = event_bus.subscribe()
263:     
264:     LOOP
265:       SELECT
266:         // Check shutdown
267:         _ = shutdown.cancelled() => {
268:           tracing::info!("Presenter {} shutting down", P::NAME)
269:           BREAK
270:         }
271:         
272:         // REQ-EVT-001.1, REQ-EVT-001.2: Receive events in order
273:         result = receiver.recv() => {
274:           MATCH result
275:             Ok(event) => {
276:               // Dispatch to presenter's handler
277:               presenter.handle_event(event).await
278:             }
279:             
280:             // REQ-EVT-001.4: Handle lagged subscriber
281:             Err(RecvError::Lagged(count)) => {
282:               tracing::warn!(
283:                 "Presenter {} lagged {} events",
284:                 P::NAME, count
285:               )
286:               // Trigger full state refresh
287:               presenter.refresh_state().await
288:             }
289:             
290:             // Channel closed
291:             Err(RecvError::Closed) => {
292:               tracing::info!("EventBus closed, presenter stopping")
293:               BREAK
294:             }
295:           END MATCH
296:         }
297:       END SELECT
298:     END LOOP
299:   })
300: END FUNCTION
301: 
302: END MODULE
```

### Test: Event Delivery to All Subscribers

```pseudocode
TEST event_bus_delivers_to_all_subscribers
  // REQ-EVT-001.1
  GIVEN EventBus with capacity 16
  AND subscriber A = bus.subscribe()
  AND subscriber B = bus.subscribe()
  AND subscriber C = bus.subscribe()
  
  WHEN bus.emit(AppEvent::System(SystemEvent::AppLaunched))
  
  THEN A.recv().await == Ok(AppEvent::System(SystemEvent::AppLaunched))
  AND B.recv().await == Ok(AppEvent::System(SystemEvent::AppLaunched))
  AND C.recv().await == Ok(AppEvent::System(SystemEvent::AppLaunched))
END TEST
```

```pseudocode

  AND fast_subscriber = bus.subscribe()
  
  WHEN bus.emit(event_1)
  AND bus.emit(event_2)
  
  THEN fast_subscriber.recv().await == Ok(event_1) // Immediate
  AND fast_subscriber.recv().await == Ok(event_2) // Immediate
  // slow_subscriber did not block anything

```

### Test: Lagged Subscriber Notification

```pseudocode
TEST lagged_subscriber_notified
  // REQ-EVT-001.4
  GIVEN EventBus with capacity 4
  AND subscriber = bus.subscribe()
  
  WHEN bus.emit(event_1) // subscriber doesn't recv
  AND bus.emit(event_2)
```

## Error Handling

| Error Condition | Handling Strategy | User Impact |
|-----------------|-------------------|-------------|
| Channel full | Log warning, use blocking send or drop | Brief delay |
| Channel closed | Log info, stop task gracefully | None (shutdown) |
| Subscriber lagged | Log warning, trigger state refresh | Potential UI flicker |
| Bridge shutdown race | Use atomic flag + channel close | Clean shutdown |

## Performance Notes

1. **Channel Capacity**: 256 events balances memory vs throughput
2. **Non-blocking Sends**: try_send prevents backpressure in UI thread
3. **Batch Commands**: send_batch reduces notification overhead
4. **Single Notification**: Notifier coalesces multiple commands
