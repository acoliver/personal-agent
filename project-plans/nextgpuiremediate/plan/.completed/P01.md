## Root Defect Identified

**REQ-WIRE-001 evidence**: The active GPUI runtime has a split event pipeline. `ChatPresenter` and `HistoryPresenter` subscribe to the global `EventBus` correctly. However, `SettingsPresenter`, `ProfileEditorPresenter`, `McpAddPresenter`, `McpConfigurePresenter`, and `ModelSelectorPresenter` subscribe to a **local** `app_event_tx` broadcast channel created in `src/main_gpui.rs` at lines 474–479 whose receiver is immediately discarded (`let (app_event_tx, _) = …`). This means `UserEvent`s for settings, profiles, MCP, and model selection are published to the global bus but the presenters for those domains never receive them — producing dead controls.

**Secondary defect**: Navigation uses a static `NavigationChannel` global (`src/ui_gpui/navigation_channel.rs`) instead of the EventBus pipeline, bypassing presenters entirely.

**Tertiary defect**: `MainPanel::handle_command()` has a `_ => {}` wildcard that silently drops all non-navigation `ViewCommand`s, meaning presenter-generated updates never reach child views.
