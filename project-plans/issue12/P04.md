# Phase 04 Evidence — Mac Native Theme (`mac-native` option)

**Phase:** 04 — Mac Native pseudo-theme: `mac-native` slug in registry, macOS AppKit color
resolver, cfg-gated non-macOS fallback, integration into `available_theme_options()` and
`Theme` color accessors.

**Date completed:** 2026-03-27

---

## Scope Summary

Phase 04 delivered:

1. **`src/ui_gpui/mac_native.rs`** — new module with:
   - `MAC_NATIVE_SLUG = "mac-native"` and `MAC_NATIVE_NAME = "Mac Native"` constants.
   - `MacNativePalette` struct containing all 13 resolved `Hsla` color tokens.
   - `#[cfg(target_os = "macos")]` platform impl using `objc2-app-kit`:
     - `is_dark_appearance()` using `NSAppearance.currentDrawingAppearance` +
       `bestMatchFromAppearancesWithNames:`.
     - `nscolor_to_rgb()` reading sRGB components via `colorUsingType:` +
       `getRed:green:blue:alpha:`.
     - `rgb_to_hsla()` converting sRGB triplet to GPUI `Hsla`.
     - `resolve_palette()` composing all 13 tokens from AppKit semantic colors.
   - `#[cfg(not(target_os = "macos"))]` stub returning `None` — callers fall back to default
     catalog theme.
   - Public `resolve_palette()` dispatcher routing to the platform-appropriate impl.

2. **`src/ui_gpui/theme.rs`** additions for Phase 04:
   - `use crate::ui_gpui::mac_native::{self, MacNativePalette, MAC_NATIVE_SLUG}` import.
   - `try_mac_native_palette()` — checks active slug and calls `mac_native::resolve_palette()`.
   - `resolve_with_mac_native()` — resolution order: mac-native palette → catalog hex → fallback.
   - Every `Theme` accessor method updated to use `resolve_with_mac_native()` with both a
     `mac_select` closure (for mac-native) and a `catalog_select` closure (for file-backed themes).
   - `available_theme_options()` appends a synthetic `mac-native` entry after all catalog entries.
   - `load_active_theme()` skips catalog lookup when slug is `MAC_NATIVE_SLUG` and falls through
     to default (used by `resolve_hex_with_fallback` fallback path).

3. **`src/ui_gpui/mod.rs`** — added `pub mod mac_native;` and `pub mod theme_catalog;`.

4. **`Cargo.toml`** — added `"objc2-core-foundation"` feature to `objc2-app-kit` dependency for
   the `colorUsingType:` API path.

5. **`tests/theme_runtime_static_tests.rs`** — 8 new Phase-04 tests (appended to existing 12
   Phase-02 tests, total 20):
   - `mac_native_slug_appears_in_available_theme_options`
   - `mac_native_entry_has_expected_name`
   - `mac_native_constants_match_expected_values`
   - `setting_mac_native_slug_is_accepted_and_reflected`
   - `mac_native_active_colors_are_valid_hsla`
   - `mac_native_fallback_is_deterministic_and_matches_default`
   - `mac_native_available_options_count_exceeds_catalog_count`
   - `unknown_slug_fallback_not_affected_by_mac_native_presence`

## 10) Semantic verification: mac-native resolution chain

**Full resolution flow for `Theme::text_primary()` when slug is `mac-native`:**

| Step | Location | Mechanism |
|---|---|---|
| 1. Accessor called | `theme.rs:Theme::text_primary()` | Calls `resolve_with_mac_native(|p| p.text_primary, |c| c.text.primary.as_str(), fallback::TEXT_PRIMARY)` |
| 2. Check active slug | `theme.rs:try_mac_native_palette()` | `get_active_slug() == MAC_NATIVE_SLUG` → `true` → calls `mac_native::resolve_palette()` |
| 3. Platform dispatch | `mac_native.rs:resolve_palette()` | Routes to `platform::resolve_palette()` |
| 4a. macOS path | `mac_native.rs:platform::resolve_palette()` | Reads `NSColor::labelColor()`, converts via `nscolor_to_rgb()` → `rgb_to_hsla()` → `Some(MacNativePalette { text_primary, ... })` |
| 4b. Non-macOS path | `mac_native.rs:platform::resolve_palette()` | Returns `None` immediately |
| 5. Palette available (macOS) | `theme.rs:resolve_with_mac_native()` | Returns `mac_select(&palette)` = `palette.text_primary` |
| 5. Palette unavailable (non-macOS / AppKit fail) | `theme.rs:resolve_with_mac_native()` | Falls through to `resolve_hex_with_fallback(|c| c.text.primary.as_str(), fallback::TEXT_PRIMARY)` which uses catalog or hard-coded fallback |

**Trace verdict: COMPLETE**

**Non-macOS compile path:** The `#[cfg(not(target_os = "macos"))]` module compiles cleanly on
all platforms. The test `mac_native_active_colors_are_valid_hsla` validates that every accessor
returns valid HSLA regardless of whether AppKit palette resolution succeeds or fails (by testing
the full range of h/s/l/a against `[0.0, 1.0]`).
