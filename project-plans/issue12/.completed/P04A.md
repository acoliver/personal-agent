## Semantic Verification

### 1. `mac-native` appears in available theme options and maps to synthetic pseudo-theme entry

**Verified.** `available_theme_options()` in `theme.rs` (lines 85–110) constructs a `ThemeOption` with `name: "Mac Native"`, `slug: "mac-native"`, `kind: Dark` and appends it after all file-driven catalog themes. Test `mac_native_slug_appears_in_available_theme_options` (line 344) asserts the slug is present. Test `mac_native_entry_has_expected_name` (line 354) asserts the name is `"Mac Native"`. Test `mac_native_available_options_count_exceeds_catalog_count` (line 469) asserts `options.len() > catalog.len()`, confirming the synthetic entry adds one entry beyond the file-driven catalog.

### 2. Runtime path when active slug is `mac-native`: token resolution attempts AppKit semantic palette on macOS; deterministic fallback otherwise

**Verified.** The resolution chain is:

1. Every `Theme` accessor calls `resolve_with_mac_native(mac_select, catalog_select, fb)` (e.g., `bg_base()` at line 332).
2. `resolve_with_mac_native` calls `try_mac_native_palette()` (line 303).
3. `try_mac_native_palette()` checks `get_active_slug() == MAC_NATIVE_SLUG` and if true calls `mac_native::resolve_palette()` (lines 137–141).
4. `mac_native::resolve_palette()` delegates to `platform::resolve_palette()` (line 233).
5. On macOS (`#[cfg(target_os = "macos")]`, line 48): `platform::resolve_palette()` reads AppKit semantic colors (`NSColor::windowBackgroundColor()`, `labelColor()`, `controlAccentColor()`, `systemRedColor()`, etc.) and converts them to `Hsla`. Returns `Some(MacNativePalette)` on success.
6. On non-macOS (`#[cfg(not(target_os = "macos"))]`, line 213): `platform::resolve_palette()` returns `None`.
7. When `resolve_palette()` returns `None`, `resolve_with_mac_native` falls through to `resolve_hex_with_fallback` which loads from the catalog (defaulting to `"default"` if `mac-native` is not in the catalog — handled at `load_active_theme()` line 118–119).
8. Test `mac_native_fallback_is_deterministic_and_matches_default` (line 443) confirms deterministic results on repeated calls.
9. Test `mac_native_active_colors_are_valid_hsla` (line 389) confirms all 13 color accessors return valid `Hsla` values with `mac-native` active.

### 3. Unknown slug fallback remains deterministic and unaffected

**Verified.** `load_active_theme()` (lines 114–125) has explicit logic: if slug is `mac-native`, it returns the `default` catalog entry (not `None`); for any other slug, it tries the catalog lookup and falls back to `"default"` via `.or_else(|| catalog.get(DEFAULT_SLUG))`. Test `unknown_slug_fallback_not_affected_by_mac_native_presence` (line 485) sets slug to `"totally-unknown-slug-99"` and asserts text_primary matches the `"default"` palette. Test `unknown_slug_falls_back_to_default_deterministically` (exists earlier in the file) also validates this independently.
