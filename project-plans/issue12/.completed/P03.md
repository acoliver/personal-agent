# Phase 03 Evidence — Settings Theme Dropdown + Event/Command Wiring

**Phase:** 03 — Settings theme dropdown, event and command plumbing for theme selection,
persistence via `AppSettingsService` set_theme/get_theme, runtime apply via ui_gpui theme
active slug setter, and startup apply from persisted setting.

**Date completed:** 2026-03-27

---

## Scope Summary

Phase 03 delivered:

1. **`UserEvent::SelectTheme { slug }`** in `src/events/types.rs` — the user-facing event fired
   when the settings dropdown selection changes.
2. **`ViewCommand::ShowSettingsTheme { options, selected_slug }`** and **`ThemeSummary`** in
   `src/presentation/view_command.rs` — the presenter→UI command carrying available themes and
   the active selection.
3. **`SettingsPresenter`** additions in `src/presentation/settings_presenter.rs`:
   - `emit_theme_snapshot()` — emits `ShowSettingsTheme` from catalog + persisted slug.
   - `on_select_theme()` — persists slug via `AppSettingsService::set_theme`, applies it to
     runtime via `set_active_theme_slug`, then re-emits the snapshot.
   - Startup `start()` calls both `emit_profiles_snapshot` and `emit_theme_snapshot`.
   - `RefreshProfiles` handler also re-emits theme snapshot.
4. **Settings view additions** in `src/ui_gpui/views/settings_view/`:
   - `mod.rs`: `ThemeOption`, `SettingsState::available_themes` / `selected_theme_slug`,
     `apply_theme_options()`, `select_theme()` method.
   - `command.rs`: `ShowSettingsTheme` arm in `handle_command` calling `apply_theme_options`.
   - `render.rs`: `render_theme_section()` and `render_theme_row()` GPUI render methods.
5. **Startup apply** in `src/main_gpui/startup.rs` — `build_startup_inputs_async` calls
   `app_settings.get_theme()` and `set_active_theme_slug()` before the first GPUI render.
