# Phase 04 Evidence — Mac Native Theme Option

## Verdict: PASS

---

## Summary

Phase 04 introduced a `mac-native` pseudo-theme that integrates with macOS AppKit semantic
color resolution at runtime. Key deliverables:

- **`src/ui_gpui/mac_native.rs`** — AppKit-backed semantic color resolver that maps
  `ThemeColors` roles (background, text, accent, border, error, surface) to macOS system
  colors via the `appkit-nscolor` crate. Includes a deterministic fallback that mirrors the
  `default` theme when AppKit colors are unavailable (e.g., in headless test environments).
- **`src/ui_gpui/theme.rs`** — Wires the `mac-native` slug into the theme catalog: the slug
  is injected into `available_theme_options()` as a synthetic entry (name "Mac Native", slug
  `mac-native`) alongside all file-driven bundled themes. `active_colors()` dispatches to the
  AppKit resolver when the active slug is `mac-native`.
- **`src/ui_gpui/mod.rs`** — Re-exports / module wiring updated so `mac_native` is visible
  within the `ui_gpui` module tree.

The fallback is deterministic: unknown slugs and the `mac-native` slug in non-AppKit
environments both resolve to the same color set as the `default` theme, making all 20
runtime-static tests deterministic under CI.
