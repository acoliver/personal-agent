## Semantic Verification

### Runtime path trace: `set_active_theme_slug` → Theme accessor → catalog lookup → fallback default

**Step 1 – `set_active_theme_slug(slug)`** (theme.rs:56-70):
Acquires write lock on `ACTIVE_THEME_SLUG: RwLock<String>`. Compares current value (treating empty string as `"default"`) against the incoming slug. If different, writes the new slug and returns `true`; otherwise returns `false`.

**Step 2 – Theme accessor (e.g., `Theme::text_primary()`)** (theme.rs:305-307):
Calls `Self::resolve_hex_with_fallback(|c| c.text.primary.as_str(), fallback::TEXT_PRIMARY)`.

**Step 3 – `resolve_hex_with_fallback`** (theme.rs:248-255):
Calls `with_active_colors(|colors| hex_str_to_hsla(select(colors)))` → `load_active_theme()`.

**Step 4 – `load_active_theme()`** (theme.rs:101-108):
1. Loads the bundled catalog via `ThemeCatalog::load_bundled()`.
2. Reads the active slug from `get_active_slug()`.
3. Looks up `catalog.get(&slug)`.
4. If the slug is not found: `.or_else(|| catalog.get(DEFAULT_SLUG))` — falls back to `"default"`.
5. Returns `Option<ThemeDefinition>` (cloned).

**Step 5 – Catalog lookup: `ThemeCatalog::get()`** (theme_catalog.rs:243-244):
`self.themes_by_slug.get(slug)` — O(1) HashMap lookup. Returns `None` for unknown slugs.

**Step 6 – Fallback on catalog load failure** (theme.rs:253-254):
If the entire catalog fails to load (missing assets dir), `with_active_colors` returns `None`, and `resolve_hex_with_fallback` falls through to `rgb_to_hsla(fb.0, fb.1, fb.2, 1.0)` using the hard-coded `fallback::*` constants.
