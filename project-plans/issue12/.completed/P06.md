## PA_FORCE_THEME Implementation Note

`PA_FORCE_THEME` is an automation-only env var added to `build_startup_inputs_async`
in `src/main_gpui/startup.rs`.  When set:
1. Logs `"Startup: PA_FORCE_THEME override active: 'â€¦'"` for diagnostics.
2. Uses the env value as the theme slug in place of the persisted setting.
3. The value is still passed through `migrate_legacy_theme_slug` so legacy aliases
   map correctly even in forced mode.

This lets automation tests deterministically screenshot each theme without writing
to or reading from the user's persistent `app_settings.json`.
