## Semantic Verification

### Trace 1: Settings Selection → Theme Apply → Snapshot Update

**End-to-end flow: user selects theme → runtime theme changes → view updates**

| Step | Location | Mechanism |
|---|---|---|
| 1. User clicks theme row in settings UI | `render.rs:563-568` | `render_theme_row()` calls `this.select_theme(slug.clone(), cx)` via `on_mouse_down` listener |
| 2. `select_theme()` updates local state and emits event | `mod.rs:205-209` | Sets `self.state.selected_theme_slug = slug.clone()`, then calls `self.emit(&UserEvent::SelectTheme { slug })`, then `cx.notify()` |
| 3. `UserEvent::SelectTheme { slug }` bridges to presenter | `mod.rs:381-389` | `emit()` calls `bridge.emit(event.clone())` which sends through the `GpuiBridge` flume channel |
| 4. Presenter dispatches `SelectTheme` | `settings_presenter.rs:241-243` | `handle_user_event()` matches `UserEvent::SelectTheme { slug }` and calls `Self::on_select_theme(app_settings_service, view_tx, slug).await` |
| 5. `on_select_theme()` persists and applies | `settings_presenter.rs:662-674` | Calls `app_settings_service.set_theme(slug.clone()).await` to persist, then `set_active_theme_slug(&slug)` to apply to runtime, then `Self::emit_theme_snapshot(...)` to update the view |
| 6. `emit_theme_snapshot()` builds and sends command | `settings_presenter.rs:636-659` | Calls `available_theme_options()` for all bundled themes, `app_settings_service.get_theme().await` for persisted slug, then sends `ViewCommand::ShowSettingsTheme { options, selected_slug }` |
| 7. Settings view receives `ShowSettingsTheme` command | `command.rs:21-26` | `handle_command()` matches `ViewCommand::ShowSettingsTheme { options, selected_slug }` and calls `self.apply_theme_options(options, selected_slug)` |
| 8. `apply_theme_options()` updates view state | `mod.rs:181-202` | Maps `ThemeSummary` → `ThemeOption`, validates slug exists in list, sets `self.state.selected_theme_slug` and `self.state.available_themes` |

**Trace verdict: COMPLETE** — the full cycle from user click through event emission, presenter handling, persistence, runtime slug application, and view command back to the settings view is wired end-to-end with no gaps.
