## Discrepancies found

1. **Cosmetic / non-blocking — spec sidebar path:** The spec's Integration Points section
   refers to `src/ui_gpui/views/render_sidebar.rs` as the primary path, with
   `src/ui_gpui/views/chat_view/render_sidebar.rs` as a parenthetical "currently lives under".
   The actual and only path is `src/ui_gpui/views/chat_view/render_sidebar.rs`. The spec
   already notes this in the parenthetical, so this is cosmetic. Implementation should use
   the actual path.

2. **Cosmetic / non-blocking — spec mentions `project_history_snapshot` function:** The spec's
   check #7 asks to grep for `project_history_snapshot` alongside `project_streaming_snapshot`.
   No `project_history_snapshot` function exists — history snapshot data is assembled inline
   in `app_store.rs`. The spec's Integration Points section says
   "`project_history_snapshot` will populate `streaming_conversation_ids`" which implies it
   either needs to be created or the inline logic needs to be extended. Not a contradiction,
   just an implementation detail for the impl phase.

3. **Expected absence — `tokio-util` not in Cargo.toml:** The spec explicitly plans for this
   ("if not already enabled"). Will be added in the impl phase.

4. **Pre-existing test issue — `test_secrets_service_crud` hangs:** This integration test
   in `tests/e2e_architecture.rs` blocks on macOS keychain access. Pre-existing,
   unrelated to issue #173. Not a blocker.
