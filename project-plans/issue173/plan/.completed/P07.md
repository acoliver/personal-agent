## Summary

This phase implements the conversation-scoped approval resolution mechanism in the `ApprovalGate` to ensure that cancelling conversation A does not wake waiters belonging to conversation B.

## Files Touched

### 1. `src/llm/client_agent.rs`

**Diff Summary:**
- Added new public method `resolve_all_for_conversation()` to the `impl ApprovalGate` block (after `resolve_all`)
- The method filters pending approvals by `conversation_id`, removes matching entries, sends the approval decision to each waiter, and returns the resolved request identifiers
- Added `@plan PLAN-20260416-ISSUE173.P07` marker
- Added `@requirement REQ-173-003.1` and `@requirement REQ-173-003.2` markers
- Refactored implementation to use two separate lock acquisitions (keys collection + removal) to satisfy clippy's `significant_drop_tightening` lint
- Changed `pending.drain().collect::<Vec<_>>()` back to original form (line 205) to maintain compatibility with original code structure

### 2. `src/services/chat_impl.rs`

**Diff Summary:**
- In `cancel_active_stream()`, replaced `self.approval_gate.resolve_all(false)` with `self.approval_gate.resolve_all_for_conversation(conversation_id, false)`
- Added `@plan PLAN-20260416-ISSUE173.P07` marker (line 233)
- Added `@requirement REQ-173-003.3` marker (line 234)
- The old `resolve_all` method is preserved for shutdown paths (not used in cancellation anymore)
