### What Was Wrong

Reviewer identified a **test logic bug** in `tests_concurrent.rs` tests 1 and 2:
- Tests started streaming on a real `GpuiAppStore` via reducer commands (`ShowThinking`, `AppendStream`)
- BUT assertions were made against a disconnected `AppStoreInner::default()` instance
- These assertions could **never** turn green — even after P09 implements the field, the tests would fail because they weren't reading from the actual store state

### What Was Fixed

1. **Added `#[cfg(test)]` accessor on `GpuiAppStore`** (`src/ui_gpui/app_store.rs`):
   - Added `pub(crate) fn active_streaming_targets_for_test(&self) -> HashSet<Uuid>`
   - Returns a clone of the internal `active_streaming_targets` field
   - Accessor references `self.inner.lock().unwrap().active_streaming_targets` — this field doesn't exist yet (P09 implements it), maintaining the RED state
   - Marked with `/// @plan PLAN-20260416-ISSUE173.P08` and `/// @requirement REQ-173-004.1`

2. **Rewrote tests 1 and 2** to use the real store accessor:
   - Test 1: After starting streams on A and B, calls `let targets = store.active_streaming_targets_for_test();` and asserts both are in the set
   - Test 2: After both begun, asserts both in; after finalize A, re-reads targets and asserts A not in and B still in
   - Test 3 kept as-is (it tests `project_streaming_snapshot` directly with manually constructed state)

3. **Removed bad import**: Removed `AppStoreInner` from imports in `tests_concurrent.rs` — no longer needed since we use the store accessor.
