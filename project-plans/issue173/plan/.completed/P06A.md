### Check 6: Semantic correctness of both tests

#### Test 1: `resolve_all_for_conversation_resolves_only_target` (line 255)

```rust
async fn resolve_all_for_conversation_resolves_only_target() {
    let gate = ApprovalGate::new();
    let conv_a = Uuid::new_v4();
    let conv_b = Uuid::new_v4();

    let waiter_a = gate.wait_for_approvals("req-a".into(), vec!["tool".into()], conv_a);
    let waiter_b = gate.wait_for_approvals("req-b".into(), vec!["tool".into()], conv_b);

    let resolved = gate.resolve_all_for_conversation(conv_a, false);
    assert_eq!(resolved.len(), 1);
    assert_eq!(resolved[0].0, conv_a);
    assert_eq!(resolved[0].1, "req-a");

    assert_eq!(waiter_a.wait().await.unwrap(), false);

    let _ = gate.resolve("req-b", true);
    assert_eq!(waiter_b.wait().await.unwrap(), true);
}
```

Verified against spec:
- [OK] Creates two waiters on different conv_ids (`conv_a`, `conv_b`)
- [OK] Calls `resolve_all_for_conversation(conv_a, false)`
- [OK] Asserts `resolved.len() == 1`
- [OK] Asserts `resolved[0].0 == conv_a` (conversation UUID matches)
- [OK] Also asserts `resolved[0].1 == "req-a"` (request ID — consistent with existing `resolve_all` return type `Vec<(Uuid, String)>`)
- [OK] `waiter_a` completes with `false`
- [OK] `waiter_b` still answerable via explicit `gate.resolve("req-b", true)` → `waiter_b.wait()` returns `true`

**Semantics: CORRECT**

#### Test 2: `resolving_one_conversation_does_not_wake_another` (line 285)

```rust
async fn resolving_one_conversation_does_not_wake_another() {
    let gate = ApprovalGate::new();
    let conv_a = Uuid::new_v4();
    let conv_b = Uuid::new_v4();

    let _waiter_a = gate.wait_for_approvals("req-a".into(), vec!["tool".into()], conv_a);
    let waiter_b   = gate.wait_for_approvals("req-b".into(), vec!["tool".into()], conv_b);

    let _ = gate.resolve_all_for_conversation(conv_a, false);

    let result = tokio::time::timeout(Duration::from_millis(50), waiter_b.wait()).await;
    assert!(result.is_err(), "waiter B should still be pending");
}
```

Verified against spec:
- [OK] Creates two waiters for two different conversations
- [OK] Uses `_waiter_a` binding (not `let _ =`) to prevent premature `Drop` which would cancel the channel
- [OK] Calls `resolve_all_for_conversation(conv_a, false)`
- [OK] Asserts `waiter_b.wait()` times out (still pending) via `tokio::time::timeout` with 50ms window
- [OK] `result.is_err()` confirms timeout occurred = waiter_b was NOT woken
