## Summary

Successfully implemented the sidebar streaming indicator feature as specified in PLAN-20260416-ISSUE173.P11:

1. **Added `streaming_conversation_ids` field to `HistoryStoreSnapshot`** (app_store_types.rs:134): A `HashSet<Uuid>` field with proper doc comments and plan/requirement markers. The `#[derive(Default)]` attribute automatically handles the `HashSet::new()` default.

2. **Populated the snapshot field in `bump_revision_and_publish`** (app_store.rs:812): Before incrementing the revision and publishing to subscribers, the history snapshot's `streaming_conversation_ids` field is populated from `inner.active_streaming_targets.clone()`. This ensures the snapshot always reflects the current set of streaming conversations.

3. **Added `conversation_has_streaming_indicator` helper** (render_sidebar.rs:20): A public helper function that checks if a conversation ID is in the streaming IDs set, returning `bool`.

4. **Added streaming indicator rendering** (render_sidebar.rs): A new `render_streaming_indicator` function creates a 6px dot using the accent color when streaming is active, or transparent when not. The indicator is rendered in `render_title_row` with a fixed-width leading slot to maintain stable layout (REQ-173-005.2/005.3).

5. **Threaded streaming IDs through the view state** (main_panel/mod.rs:93-96, chat_view/state.rs:82-86): The `streaming_conversation_ids` set flows from the store snapshot through the main panel to the chat view state, where it's used by the sidebar render functions.

6. **Updated test code** (history_view.rs): Added the new field to all test HistoryStoreSnapshot constructions.
