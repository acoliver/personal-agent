## 2. Summary of Key Design Decisions

### 2.1 IR Type Structure (from ir-type-design.md, now encoded in pseudocode)

**MarkdownInline** (Lines 399-405 in parse-markdown-blocks.md):
- Fields: `text`, `bold`, `italic`, `strikethrough`, `code`, `link_url`
- Style flags are boolean for composability (bold + italic + link combinations)

**MarkdownBlock Variants** (Lines 25-196 in parse-markdown-blocks.md):
- `Paragraph { spans, links }` - with per-block link storage as `(Range<usize>, String)`
- `Heading { level, spans, links }` - same link structure as Paragraph
- `CodeBlock { language, code }` - raw text, not parsed as markdown
- `BlockQuote { blocks }` - recursive Vec<MarkdownBlock>
- `List { ordered, start, items }` - items is Vec<Vec<MarkdownBlock>>
- `Table { alignments, header, rows }` - TableCell has spans + links
- `ThematicBreak` - empty variant
- `ImageFallback { alt }` - not rendering actual images per spec

### 2.2 Three-Stack Architecture (Lines 8-19 in parse-markdown-blocks.md)

The parser uses three concurrent stacks:
1. **Block stack** (`Vec<BlockBuilder>`) - tracks block-level nesting (Paragraph, Heading, CodeBlock, etc.)
2. **Inline style stack** (`Vec<InlineStyleState>`) - tracks bold/italic/strikethrough/link nesting
3. **List stack** (`Vec<ListContext>`) - tracks ordered/unordered list state for bullet/number generation

### 2.3 Link Range Tracking (Lines 251-274 in parse-markdown-blocks.md)

Byte range calculation for GPUI InteractiveText:
- `link_start_offset` captured at Start(Link)
- `link_end_offset` calculated at End(Link) using `count_bytes_in_spans()` helper
- Stored as `(Range<usize>, String)` tuple in block's `links` field
- Conversion from char positions (pulldown-cmark) to byte offsets (GPUI) handled in helper

### 2.4 HTML Tag Stripping (Lines 414-441 in parse-markdown-blocks.md)

Simple state machine approach:
- `in_tag` boolean tracks whether inside `<...>`
- Characters inside tags dropped; outside tags appended
- Malformed HTML (unclosed `<`) degrades to literal text per REQ-MD-PARSE-050
- Both `Html` (block-level) and `InlineHtml` (inline) events handled

### 2.5 TextRun Construction (Lines 383-449 in blocks-to-elements.md)

Style stacking for GPUI TextRun:
- Font family: "Menlo" for code, system default otherwise
- Font weight: BOLD if `span.bold`
- Font style: Italic if `span.italic`
- Underline: Some() if `span.link_url.is_some()`
- Strikethrough: Some() if `span.strikethrough`
- Background: Some(Theme::bg_darker()) if `span.code`
- Text color: accent() for links, text_primary() otherwise

### 2.6 Click Event Precedence (Lines 173-189 in render-markdown.md)

Integration decision matrix implemented in pseudocode:
- No links + not streaming: attach `.cursor_pointer()` + `.on_click()` for copy
- Has links: no bubble click handler (InteractiveText handles clicks)
- Streaming: no click handler (cursor interaction reserved)

### 2.7 URL Security (Lines 59-82 in render-markdown.md)

Scheme allowlist implementation:
- Only "https" and "http" schemes permitted
- Uses `url::Url::parse()` for RFC 3986 compliance
- Malformed URLs and rejected schemes return false silently (REQ-MD-SEC-006)
