## Implementation Bugs Found and Fixed

### 1. BM25 Aggregate Bug (conversation_sqlite.rs `search` method)

**Root cause:** The spec's CTE query used `MIN(bm25(search_index, 10.0, 1.0))` inside a
`GROUP BY` clause. SQLite FTS5 auxiliary functions (`bm25`, `snippet`) cannot be used as
aggregate arguments — they require the FTS virtual table to be the driving query with no
aggregation layer. This caused `"unable to use function bm25 in the requested context"` errors.

**Fix:** Rewrote the search query to collect one row per FTS hit (no `GROUP BY`), then
aggregate per `conversation_id` in Rust using a `HashMap`. The best score and context snippet
are selected in Rust, then results are sorted and paginated. Produces identical ranking semantics
to the spec query.

### 2. QueryReturnedNoRows → NotFound Mapping (db/worker.rs)

**Root cause:** `DbHandle::execute` mapped all `rusqlite::Error` variants uniformly to
`ServiceError::Storage(...)`. When `query_row` returned `QueryReturnedNoRows` (e.g., after
deleting a conversation and then loading it), the service returned `Storage("Query returned no rows")`
instead of `NotFound(...)`. The per-method `map_err` checks in `load`, `delete`, etc. correctly
tested for `ServiceError::NotFound`, but it was never produced by the execute pathway.

**Fix:** Added pattern match in `DbHandle::execute` to map `rusqlite::Error::QueryReturnedNoRows`
to `ServiceError::NotFound("record not found")` before the catch-all `Storage` variant.

### 3. blocking_recv in Tokio Context (pre-existing in test suite)

**Root cause:** `spawn_db_thread` calls `blocking_recv()` internally for its initialization
handshake. When called directly from a `#[tokio::test]` async context, this panics because
tokio's worker threads cannot block.

**Fix:** All `spawn_db_thread` call sites in tests and `App::new` now use
`tokio::task::spawn_blocking(move || spawn_db_thread(&path))`. Fixed in:
- `tests/sqlite_conversation_service_tests.rs` (`make_service` helper)
- `tests/config_storage_tests.rs`
- `tests/service_coverage_tests.rs` (2 tests)
- `tests/e2e_architecture.rs`
- `src/app.rs` (`initialize_services`)
