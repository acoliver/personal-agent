# P01: DB Infrastructure ‚Äî Completed

## Phase Summary

Phase 01 implements the SQLite database infrastructure layer for issue #56 (SQLite Conversation Storage with FTS5 Search).

## Deliverables

### 1. `Cargo.toml` ‚Äî rusqlite dependency added

```toml
rusqlite = { version = "0.33", features = ["bundled"] }
```

### 2. `src/db/mod.rs` ‚Äî public module exports

Re-exports `DbHandle` and `spawn_db_thread` from `worker`, and exposes `schema` as a public submodule.

### 3. `src/db/schema.rs` ‚Äî DDL + `initialize_schema`

- Applies 4 PRAGMAs on every connection open: WAL, foreign_keys, busy_timeout=5000, synchronous=NORMAL
- Checks `PRAGMA user_version`:
  - If 0: runs full DDL (conversations, messages, search_index FTS5, all 5 triggers, all 3 indexes), then sets `user_version = 1`
  - If 1: no-op
- All tables/indexes use `IF NOT EXISTS` for safety
- All 5 triggers present: `messages_ai`, `messages_au`, `messages_ad`, `conversations_title_au`, `conversations_ad`
- FTS5 virtual table `search_index` with `title`, `content`, `conversation_id UNINDEXED`, `message_rowid UNINDEXED`

### 4. `src/db/worker.rs` ‚Äî DB worker thread

- `type DbJob = Box<dyn FnOnce(&rusqlite::Connection) + Send + 'static>`
- `pub struct DbHandle { sender: std::sync::mpsc::Sender<DbJob> }`
- `DbHandle::execute<F, R>` async method ‚Äî sends closure via channel, awaits oneshot result
- `DbHandle: Clone` ‚Äî clones the `mpsc::Sender`
- `pub fn spawn_db_thread(db_path: &Path) -> Result<DbHandle, ServiceError>`:
  - Spawns OS thread named "db-worker"
  - Opens connection via `rusqlite::Connection::open`
  - Initializes schema, reports success/failure via oneshot channel
  - Runs `rx.recv()` loop with `catch_unwind` around each job
  - Returns `DbHandle` or `ServiceError`

### 5. `src/lib.rs` ‚Äî `pub mod db` added
