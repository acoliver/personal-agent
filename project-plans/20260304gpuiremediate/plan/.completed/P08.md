## Changes Summary

### MainPanel Simplification (`src/ui_gpui/views/main_panel.rs`)
- Removed `startup_commands: Option<Vec<ViewCommand>>` from `MainPanelAppState`

- `MainPanelAppState` construction no longer includes `startup_commands`

## Remaining MainPanel Responsibilities (bounded enumeration)
1. **Popup lifecycle**: mount/unmount handling
2. **Navigation**: `NavigationState` routing between Chat/History/Settings views
3. **Child-view composition**: creates ChatView, HistoryView, SettingsView in init
4. **Store subscription**: `subscribe_to_store_snapshots()` → `apply_store_snapshot()`
5. **Redraw notification**: `cx.notify()` on snapshot change
6. **Settings/streaming forwarding**: `handle_command()` for commands not yet fully store-driven
7. **Runtime refresh requests**: `request_runtime_snapshots()` for settings initial data

NOT a MainPanel responsibility:
- ~~Transcript ownership~~ (store-authoritative)
- ~~Startup command replay~~ (removed)
- ~~Bridge draining~~ (no-op)
- ~~Selection authority~~ (selection intent channel → store)
