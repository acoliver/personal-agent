## Changes Summary

### Protocol Migration (ViewCommand contract change)
- `src/presentation/view_command.rs`: `ConversationActivated` and `ConversationMessagesLoaded` now carry `selection_generation: u64`; added `ConversationLoadFailed { conversation_id, selection_generation, message }`
- `src/events/types.rs`: `UserEvent::SelectConversation` now carries `selection_generation: u64`
- All constructors, match arms, forwarding sites across `src/` and `tests/` updated consistently

### GPUI-Owned Selection Boundary
- `src/ui_gpui/selection_intent_channel.rs`: `request_select`/`take_pending` API for child views to request selection without minting generations
- `src/main_gpui.rs`: `handle_select_conversation_intent(...)` calls `begin_selection()` on store, emits to bridge with minted generation, immediately reduces `ConversationLoadFailed` on synchronous bridge emit failure
- `src/main_gpui.rs`: `drain_selection_intents(...)` loops the intent channel
- `src/ui_gpui/views/chat_view.rs`: uses `selection_intent_channel().request_select(...)` instead of emitting `UserEvent::SelectConversation`
- `src/ui_gpui/views/history_view.rs`: uses `selection_intent_channel().request_select(...)` instead of emitting `UserEvent::SelectConversation`

### App-Root Bridge Pump
- `src/main_gpui.rs`: `spawn_runtime_bridge_pump(...)` — sole production `drain_commands()` owner, runs at 16ms cadence via app-root `cx.spawn(...)`, stays alive with zero mounted popups
- Each tick: drain selection intents → drain bridge commands → clone batch → reduce through store → publish snapshot if changed → forward command batch to MainPanel for non-store routing
- `src/main_gpui.rs`: `publish_store_snapshot_to_main_panel(...)` pushes store snapshots to mounted MainPanel
- `src/main_gpui.rs`: `forward_runtime_commands_to_main_panel(...)` forwards full command batch through popup window for non-chat command routing (settings, MCP, profiles, etc.)
- `src/ui_gpui/views/main_panel.rs`: `ensure_bridge_polling(...)` converted to no-op; app-root pump owns bridge draining

### Store Authority (from Phase 04 skeleton → Phase 05 runtime authority)
- `src/ui_gpui/app_store.rs`: `begin_selection(...)` is the sole generation minting site
- `src/ui_gpui/app_store.rs`: `reduce_batch(...)` with generation-aware reducer for `ConversationActivated`, `ConversationMessagesLoaded`, `ConversationLoadFailed`, streaming, thinking, and transcript commands
- Same-id reselection while Loading/Ready is no-op; from Error retries with new generation
- Stale/no-op commands don't bump revision or publish

### ChatView Store Snapshot Application
- `src/ui_gpui/views/chat_view.rs`: `apply_store_snapshot(...)` now applies transcript, streaming state, and loading state from store snapshot
- Ready → applies full transcript; Loading → preserves existing messages (no clear); Error → preserves messages, reflects error

### Presenter Generation Threading
- `src/presentation/chat_presenter.rs`: `handle_select_conversation(...)` consumes enriched `selection_generation`, emits generation-aware `ConversationActivated`, `ConversationMessagesLoaded`, and `ConversationLoadFailed`
