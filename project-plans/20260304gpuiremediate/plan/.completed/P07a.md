### Same store handle identity survives across unmount/remount
Test 3 proves `Arc::ptr_eq(&original, &clone)` — same allocation. Mutation via one reference is visible via the other.

### Remount reads already-current snapshot without replay
Test 2 subscribes after mutation (simulating popup reopen) and reads `current_snapshot()` — it shows the transcript delivered while popup was absent.

### Negative control
If a popup-local proxy held transcript state independently, `current_snapshot()` after remount would show stale data. Tests 1-3 would fail.
