## Structural Verification

- `src/ui_gpui/app_store.rs` exists and defines a real store skeleton plus snapshot slices:
  - `ChatStoreSnapshot`
  - `HistoryStoreSnapshot`
  - `SettingsStoreSnapshot`
  - `GpuiAppSnapshot`
  - `GpuiAppStore`
- `src/ui_gpui/mod.rs` exports `pub mod app_store;` and re-exports `GpuiAppSnapshot` / `GpuiAppStore`.
- `src/main_gpui.rs` now constructs one process-lifetime `Arc<GpuiAppStore>` during runtime startup via `GpuiAppStore::from_startup_commands(startup_commands.as_deref())`, stores it on global `AppState`, and threads the same handle into `MainPanelAppState` for popup reopen.
- `src/ui_gpui/views/main_panel.rs` now has real snapshot plumbing:
  - `store_snapshot_revision`
  - `_store_subscription_task`
  - `apply_store_snapshot(...)`
  - `ensure_store_subscription(...)`
  - startup-time `app_store.current_snapshot()` application after replaying `startup_commands`
- `src/ui_gpui/views/chat_view.rs` now exposes `apply_store_snapshot(...)` for store-backed selected conversation id/title + dropdown sync.
- `src/ui_gpui/views/history_view.rs` now exposes `apply_store_snapshot(...)`, keeps `selected_conversation_id` in `HistoryState`, and rebuilds/render-highlights `ConversationItem.is_selected` from store snapshot data.
- Required `@plan`, `@requirement`, and `@pseudocode` markers exist across touched production files.

## Semantic Verification

- Store lifetime is no longer popup-local:
  - `AppState` now owns `app_store: Arc<GpuiAppStore>`.
  - `open_popup()` reuses the same `app_store` handle through `MainPanelAppState` instead of constructing a popup-local store.
- MainPanel is no longer the only durable owner of startup-visible selection state:
  - after legacy bootstrap replay, `MainPanel::apply_startup_commands()` deterministically applies `app_store.current_snapshot()`.
  - mounted views therefore consume store-backed selected conversation id/title/highlight state on an active startup path.
- The minimum required Phase 04 authority transfer occurred on an active path:
  - `snapshot_from_startup_commands(...)` seeds `chat.selected_conversation_id`, `chat.selected_conversation_title`, and `history.selected_conversation_id` from legacy startup commands.
  - `ChatView::apply_store_snapshot(...)` applies store-backed selected conversation id/title.
  - `HistoryView::apply_store_snapshot(...)` plus `render_conversation_card(...)` applies store-backed selection/highlight styling.
- Phase boundary is preserved:
  - startup transcript durability still remains on `startup_commands` / `apply_startup_commands()`.
  - runtime presenter/bridge reduction still remains on the legacy popup-bound path.
  - no `selection_generation`, `ConversationLoadFailed`, or reducer freshness routing was introduced yet.
- The still-red tests are the carried-forward expected-red Phase 03 failures, not new Phase 04 regressions:
  - `startup_and_manual_selection_do_not_yet_converge_on_one_authoritative_delivery_path`
  - `selection_generation_protocol_is_not_present_yet`
  - `manual_selection_still_clears_local_transcript_before_runtime_replay_arrives`
- Placeholder detection for the whole repo-wide grep is noisy because untouched UI components use legitimate `placeholder` identifiers/comments unrelated to Phase 04. Follow-up grep on the actual Phase 04 touched files returned no `todo!`, `unimplemented!`, TODO/FIXME/HACK comments, or placeholder-style markers.
