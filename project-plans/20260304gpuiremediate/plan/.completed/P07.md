## Anti-Mirror Proof Artifact

### Dropped popup-local objects
When popup closes, `MainPanel` is dropped, which drops `_store_subscription_task` (a GPUI spawn handle). This drops the `flume::Receiver<GpuiAppSnapshot>` held in the async task. The corresponding `flume::Sender` in the store's subscriber list becomes disconnected.

### Popup-absence witnesses (both required)
1. **Subscription plumbing witness**: `store.subscriber_count() == 0` after prune — proven in tests 1 and 2
2. **Teardown assertion**: receiver dropped, `flume::Sender::is_disconnected()` returns true — proven by subscriber_count dropping after receiver drop

### Surviving store handle identity
`Arc::ptr_eq(&original, &clone) == true` — proven in test 3. Same allocation, same store. No popup-local mirror.

### Production ingress during popup absence
`spawn_runtime_bridge_pump(...)` in `main_gpui.rs` — launched from app root `cx.spawn(...)`, independent of popup mount. Proven by source readback in test 5.

### Runtime ingress is NOT popup-retained proxying
`ensure_bridge_polling` in `main_panel.rs` is a documented no-op. No production `drain_commands()` in popup-bound code. Proven by source readback in test 5.
