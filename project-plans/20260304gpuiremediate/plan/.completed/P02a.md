## Semantic Verification

- The contract is singular and authoritative. It explicitly roots lifetime/ownership in app-root GPUI runtime state, names popup/window lifetime as subscriber-only, and forbids split write authority across popup-local state and the store.
- The loading protocol is explicit and generation-aware. It defines `Idle`, `Loading`, `Ready`, and `Error`, requires generation matching on success/failure, ignores stale payloads, forbids unconditional clear-on-activation, and makes same-id reselection behavior deterministic.
- MainPanel is no longer the semantic transcript owner in the target design. It is defined as a thin renderer/composition root over snapshots, with direct command forwarding barred as the authority path after Phase 05.
- The analysis set explicitly names each required semantic owner:
  - runtime-ingress owner: app-root runtime state and `spawn_runtime_bridge_pump(...)` as sole production drainer
  - single minting site: `begin_selection(...)` invoked from `handle_select_conversation_intent(...)`
  - startup visibility model: startup batch commits before any popup subscriber exists; pre-subscription publication is silent; first subscriber reads `current_snapshot()`
  - finalize authority: store reducer handles `FinalizeStream` as the durable finalization path
  - bounded-clear owner: mounted clear-handling path may only do same-turn readback from authoritative `current_snapshot()` without mutating store authority or creating a mirror
- The startup/runtime convergence is concrete rather than aspirational. The contract gives one canonical startup transaction entrypoint, forbids second startup mutators for selected-conversation state, and constrains Mode B loading to explicit seam-class reasons.
- No alternate semantic owner or second startup mutation path is left available by the written contract. Remaining compatibility glue is explicitly bounded and non-authoritative.

## Evidence Summary

The analysis set is coherent enough for downstream implementation without guessing semantic ownership. It establishes one durable authoritative store, one ordinary-runtime minting boundary, one startup transaction boundary, one app-root runtime ingress pump, explicit generation-aware loading states, explicit non-authoritative handling of `ConversationActivated`, durable reducer ownership of `FinalizeStream`, and a tightly bounded `ConversationCleared` compatibility rule. MainPanel is reduced to snapshot subscription/rendering and intent routing rather than transcript authority. On this basis, Phase 02a verification passes.
