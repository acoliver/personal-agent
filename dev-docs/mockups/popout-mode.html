How do I handle cancellation in tokio tasks?
claude-sonnet-4-20250514
There are several approaches to cancellation in tokio. The most common pattern uses tokio::select! with a cancellation token. The key insight is that dropping a future in Rust cancels it, so select! gives you cooperative cancellation by racing your task against a signal.
Can you show me a concrete example?
claude-sonnet-4-20250514
Sure! Here's a pattern using CancellationToken from tokio_util:

let token = CancellationToken::new();
let cloned = token.clone();

tokio::spawn(async move {
  select! {
    _ = cloned.cancelled() => {}
    result = do_work() => {
      handle(result);
    }
  }
});

// Cancel the task later:
token.cancel();
This pattern is clean and composable...
