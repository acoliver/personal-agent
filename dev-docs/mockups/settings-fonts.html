6. Keyboard Shortcuts — Font Zoom
Font Zoom (works from any view)
⌘+ / ⌘=  →  Increase base font size by 1px
⌘-  →  Decrease base font size by 1px
⌘0  →  Reset to default (14px)
Behavior
• Affects the same "Font Size" setting in Appearance
• Persisted to app_settings immediately
• Range: 10px – 24px (clamped)
• All text scales: body, headings (proportional), UI labels, code
• ⌘= and ⌘+ both zoom in (standard macOS behavior)
What Scales
base_size   = user setting (default 14)
H1   = base × 2.0
H2   = base × 1.5
H3   = base × 1.25
H4   = base × 1.0 (bold)
code   = base × 0.9
UI     = base × 0.85
small = base × 0.78
⌘+ and ⌘- are global key bindings registered at the window level.
They modify the persisted font_size setting and trigger a UI refresh.
The stepper in Appearance > Font Size reflects the same value.
7. Settings Model — app_settings Keys
Persisted Settings
{ "font_size": 14, "ui_font_family": null, "code_font_family": "Menlo", "code_ligatures": true }
Key Descriptions
font_size — Base font size in px (10–24, default 14). Single value drives all scaling.
ui_font_family — Font name for chrome/UI or null for system default.
mono_font_family — Font name for backtick/fenced content. Default "Menlo".
mono_ligatures — Enable OpenType calt feature for mono font. Default true.
Theme::font_size() (Proposed)
impl Theme { fn base_font_size() -> f32 { /* read from global */ } pub fn font_size_h1() -> f32 { base * 2.0 } pub fn font_size_h2() -> f32 { base * 1.5 } pub fn font_size_h3() -> f32 { base * 1.25 } pub fn font_size_body() -> f32 { base } pub fn font_size_code() -> f32 { base * 0.9 } pub fn font_size_ui() -> f32 { base * 0.85 } pub fn font_size_small() -> f32 { base * 0.78 } }
Replaces the current FONT_SIZE_XS/SM/MD/BASE/LG constants
with runtime-backed methods that read the persisted setting.
Only 4 persisted keys. Body font family is intentionally omitted —
body text uses the UI font. This keeps settings minimal. If users request
separate body font control, it's one more dropdown + key to add later.
